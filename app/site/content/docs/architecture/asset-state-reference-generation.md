# 资产状态参考生成

剧本抽取把角色、场景和道具建模为“实体基础 + 可见状态”。状态只在年龄、身份、妆发、服装、空间结构、长期陈设变化、损耗或破损等会改变画面一致性的条件发生变化时创建。

一次性的日夜、晨昏、普通天气、轻微光线变化、姿势、表情或摆放方向默认不创建状态资产。它们由镜头画面提示词修正，继续复用基础参考图。拆分不确定时默认复用基准资产，避免产生只使用一次且无法维持一致性的孤立设定图。

每个实体家族恰有一个基准状态。其余状态记录为派生状态，并保存所依赖的基准资产名称。角色状态共用同一人物基础；场景状态共用同一空间结构和材质体系；道具状态共用同一物件本体、尺寸和关键细节。

造型页会标注每张资产卡被哪些镜头画面使用。删除派生状态时，服务端会先将其镜头角色、场景、道具关联，以及已确认的提取候选改为对应基准资产，再删除该派生状态；同一镜头已有关联基准时会合并重复关联。若基准资产不存在，删除会被拒绝，避免后续画面生成失去参考图。

设定页批量生成按依赖顺序执行：先生成基准图，再生成派生状态。派生任务在实际执行时读取基准图文件并作为参考图提交。若基准图缺失或生成失败，派生任务失败，不会退化为无参考图的独立生成。

镜头拆解使用状态资产的完整名称进行关联，因此镜头帧生成会引用当前剧情所需的角色、场景和道具状态，而不是仅引用实体基础名。

道具只在当前镜头的画面、动作或构图中被明确使用时才建立镜头关联。画面生成会把该道具的基准图作为参考图提交，保持外形、尺寸、材质和关键细节一致；未出现的道具不会因为属于同一项目而被带入画面。旧项目没有保存关联时，前端仅以镜头文本中的精确名称作兼容匹配。

照片、手机屏幕、显示器、相框、海报等承载可见画面内容的道具可记录强关联角色和场景。生成该道具时，系统会把这些实体的造型图一并作为参考图提交，使屏幕或照片中的人物、地点与项目设定一致。普通物件不会建立这种关联，避免把无关人物或地点强加到道具图中。

造型页和分镜页的批量生成均可主动停止。停止会取消正在执行的图片任务，并把尚未开始的项目标记为已停止；已经成功生成的图片会保留。队列状态保存在服务端，用户刷新或离开页面不会使队列停止，返回相应页面后会继续显示进度。

画面工作台支持批量导出当前项目已有的关键帧。导出 ZIP 只读取已生成图片，不会触发生成；文件按“第几集_第几镜_镜头标题”命名，便于交付与剪辑整理。
